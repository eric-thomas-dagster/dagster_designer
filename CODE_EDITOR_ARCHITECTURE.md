# Code Editor & Custom Components Architecture

## Overview

Extending the designer to support **all Dagster capabilities**, not just declarative components:

1. **In-Browser Code Editor** (like GitHub Codespaces)
2. **Custom Component Templates** for Dagster primitives
3. **Multi-View UI** (Assets, Schedules, Jobs, Sensors, Checks)
4. **Integration Library** for all Dagster integrations

---

## 1. Code Editor Integration

### Why We Need It

**Problem:** Some integrations require Python code:
- Custom Python assets
- SQL assets with templating
- Complex sensors logic
- Custom asset checks
- Integration-specific code (AWS Lambda, Databricks notebooks, etc.)

**Solution:** Embedded Monaco Editor (VS Code's editor)

**File Browser:**
- Tree view of project files
- Create/delete files and folders
- File search
- Git status indicators

**Code Editor:**
- Syntax highlighting (Python, YAML, SQL)
- IntelliSense/autocomplete
- Error highlighting
- Format on save
- Multi-file editing (tabs)

**Terminal:**
- Run Dagster commands
- Execute Python scripts
- View logs
- Interactive Python REPL

- Asset preview updates as you type
- Syntax validation
